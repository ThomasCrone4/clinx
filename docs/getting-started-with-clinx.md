# Getting Started with Clinx

Clinx helps schools identify emerging concerns earlier by learning from patterns in the data schools already hold. Our onboarding approach is designed to be practical, light-touch, and low effort for school staff.

> The goal is simple: use existing school data to support earlier review and intervention, without asking teachers to maintain another monitoring system.

## At a Glance

| Area | What to Expect |
| --- | --- |
| Typical onboarding model | Clinx-led, with light input from the school |
| Typical starting point | Historical CSV exports first, then live or scheduled feeds where appropriate |
| Delivery approach | Phased and collaborative, with timing agreed based on school scope and data readiness |
| Main school contacts | Usually one implementation lead, one MIS/data contact, and one pastoral or safeguarding lead |
| Teacher workload | Minimal; no separate daily data entry should be required |

## How Onboarding Works

```text
Request Demo / Discovery
        |
        v
Agree Scope and Systems
        |
        v
Secure Historical Data Handover
        |
        v
Data Mapping and Validation
        |
        v
Staff Setup and Alert Preferences
        |
        v
Pilot Go-Live and Early Review
```

### Step 1. Discovery and Scope

We agree the starting scope together, including:

- the school systems currently in use
- the year groups or phases to include first
- the key school contacts for implementation
- the preferred rollout approach, usually phased rather than all-at-once

### Step 2. Historical Data Handover

Clinx usually starts with historical exports from the systems the school already uses. This allows us to:

- understand existing attendance, behaviour, timetable, and concern patterns
- train and validate the initial model against past outcomes
- prepare a school-specific starting point before wider rollout

### Step 3. Data Mapping and Validation

Clinx handles the technical work of aligning and validating the data. During this stage we:

- check identifiers and record quality
- match the data into a consistent structure
- review whether the outputs make sense with school leaders
- confirm that alerts and insight thresholds feel appropriate

### Step 4. Staff Setup

Before go-live, Clinx helps set up:

- school alert routing
- leadership visibility
- teacher alert preferences
- the initial user accounts needed for the pilot

### Step 5. Pilot Go-Live

The school begins using Clinx with a small number of users or a defined rollout group. During the early review period, Clinx supports:

- feedback on alert usefulness
- minor adjustments to routing and preferences
- review of emerging patterns and cohort-level signals

## How Timing Is Agreed

Clinx does not use a one-size-fits-all rollout timetable. The right pace depends on:

- which systems are in scope
- how much historical data is available
- the format and quality of the data shared
- whether the school wants to begin with a focused pilot or a broader rollout
- the amount of validation needed before go-live

For early partner schools, onboarding is likely to be more hands-on and more iterative while the model and implementation approach are refined together.

The most accurate way to describe timing is:

> Clinx agrees a phased onboarding plan with each school after discovery, rather than promising a fixed rollout window in advance.

## Data Usually Needed From Your School

Clinx is designed to work with data the school already collects.

### Core Data Sources

| Source | Examples of Data | Why It Helps |
| --- | --- | --- |
| MIS / Arbor-style data | pupil identifiers, year group, form, attendance history, enrolment data, timetable context | provides the core school and attendance picture |
| Behaviour data / Class Charts-style data | behaviour events, homework patterns, class-level incidents, teacher context | helps identify changes in engagement and behaviour patterns |
| Historical concern data / CPOMS-style exports where available | past concern records, chronology, category, dates, linked context | helps train the model against later recorded concerns |
| Staff and class context | teacher, class, subject, and pastoral structure | helps Clinx surface who may need to act and where concern is clustering |

### Helpful Starting Point

For most schools, a strong starting point is:

- 12-24 months of historical attendance data
- 12-24 months of behaviour or classroom incident data
- historical concern or chronology data where available
- current pupil, class, and timetable context

If everything is not available on day one, Clinx can still begin with a practical first phase and expand over time.

### Preferred Formats

Clinx can start with:

- CSV exports
- scheduled exports
- live or scheduled API-fed data where available

In most cases, the best early approach is:

1. historical CSV exports for setup and validation
2. live or scheduled feeds added over time where useful

### What Schools Do Not Need To Do

Schools should not need to:

- ask teachers to complete new daily forms
- maintain a second manual monitoring spreadsheet
- re-enter data already held elsewhere
- run complex technical setup on their own
- wait for every system to have live API access before getting started

## Who Is Usually Involved

Clinx onboarding usually only needs a small group from the school side:

- one school implementation lead
- one MIS or data contact
- one pastoral, safeguarding, or wellbeing lead

Teachers should not need to be heavily involved in setup.

## Common School Questions

### Do we need live API access to get started?

No. A CSV-first onboarding approach is often the most practical starting point.

### Do we need to change our existing systems?

No. Clinx is designed to work alongside the systems the school already uses.

### Will this create extra workload for staff?

It should create very little extra workload. Clinx is intended to reduce the amount of manual cross-checking school staff need to do.

### What if our data is not perfect?

That is normal. Clinx works with the school to validate and improve the starting dataset during onboarding.

## What Success Looks Like Early On

In the first phase, a successful rollout usually means:

- the right people can access the right insight
- alerts are routed sensibly
- the school can see which year groups, classes, or pupils may need earlier review
- teachers are not being asked to create extra records just to use the system
